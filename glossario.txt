Cada entidade do core que eu preciso consulta/salvar precisa de uma interface para implementar a ação. Essa interface é feita por meio do repository.
Essas interfaces são os contratos (ports): dizem “o que” o domínio precisa; a infra depois fornece o “como”.


usecase:
Use case (caso de uso) é a unidade de trabalho da sua aplicação: uma operação completa que atende um objetivo do usuário/sistema e orquestra tudo que precisa acontecer para isso.
Ex.: “Marcar agendamento”, “Confirmar agendamento”, “Cadastrar tutor”.

O que é um Use Case

Ação de negócio completa (verbo): orquestra entidades + políticas + repositórios para cumprir um objetivo.
Onde vive: camada application.
Faz: valida regras de aplicação, consulta/salva via contratos, decide fluxos e estados.
Não faz: SQL, HTTP, UI, detalhes de infra.

O que é Repository/Gateway

Contrato de acesso a algo externo ao domínio:
Repository → persistência (banco): find/save/list/exists…
Gateway → serviço externo (ex.: WhatsApp, pagamentos)
Onde vive: interface no core (domínio); implementação na infra.
Faz: somente ler/gravar/consultar; não aplica regra de negócio.

Regra mental rápida

Se a operação é “pegar/guardar dados” → Repository/Gateway.
Se a operação é “fazer algo de negócio” (com passos e decisões) → Use Case.
